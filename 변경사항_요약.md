# 트라이어드 / 포 노트 보이싱 분리 — 변경 요약

## 🎯 설계 원칙

| 섹션 | 구성 체계 | 학습자 관점 |
|---|---|---|
| **트라이어드** | String Set(1-3, 2-4, 3-5, 4-6)별 Root / 1st / 2nd 전위 | **음 쌓는 순서** 학습 |
| **포 노트 보이싱** | 루트 선(6/5/4/3번선)별 R-3-5-7 인터벌 배치 | **인터벌 관계** 학습 |

두 섹션은 목적이 다르므로 **완전히 분리된 자료 구조**와 **학습 흐름**을 갖도록 재구성했습니다.

---

## ✂️ 주요 변경 사항

### ① `TRIAD_VOICINGS`에서 7th 코드 제거
- 제거된 항목: `'7'`, `'maj7'`, `'m7'`, `'m7b5'` (약 90줄)
- 트라이어드는 **3화음 전용**: `major` / `minor` / `dim` / `aug`
- 각 타입별 `Root` / `1st Inv` / `2nd Inv` 3개 전위만 유지

### ② 7th 모드 전환 UI 로직 제거
- 삭제: `_is7thChord()` 함수
- 삭제: `_sync7thUI()` 함수 (버튼 라벨 동적 변경, 456 그룹 숨김, 3rd Inv 토글 등 복잡한 분기)
- `renderTriadDiagram()`에서 `is7th` 관련 분기 제거 → 코드 단순화

### ③ 포 노트 보이싱 섹션 7th 전담화
**`FOUR_NOTE_TYPES` 재정의:**
```js
{
  'maj7':  [0, 4, 7, 11],   // R · 3 · 5 · 7
  'dom7':  [0, 4, 7, 10],   // R · 3 · 5 · ♭7
  'm7':    [0, 3, 7, 10],   // R · ♭3 · 5 · ♭7
  'm7b5':  [0, 3, 6, 10],   // R · ♭3 · ♭5 · ♭7  (하프 디미니시)
  'dim7':  [0, 3, 6,  9],   // R · ♭3 · ♭5 · °7  (풀 디미니시)
}
```

- **제거**: `major`, `minor` (트라이어드에 속함 → 중복 방지)
- **명명 통일**: 기존 `'dim'`을 `'dim7'`로 개명 (실제로는 풀 디미니시이므로)
- **기본값 변경**: `_fourNoteType = 'maj7'` (이전: `'major'`)

### ④ 학습 보조: 인터벌 역할 범례
**요청하신 R/3/5/7 색상 가이드를 동적 생성:**
- 현재 선택한 코드 타입에 **실제로 포함된 인터벌만** 표시
- 예: `maj7` 선택 시 → `Root / 3 / 5 / 7` 표시
- 예: `m7♭5` 선택 시 → `Root / ♭3 / ♭5 / ♭7` 표시
- DOM 엘리먼트 `#fournote-legend`에 렌더링

### ⑤ 학습 보조: 루트 선별 보이싱 "공식" 힌트
요청하신 규칙을 `FN_VOICING_RULES`에 명시적으로 인코딩:

| 루트 선 | 배치 | 생략 |
|---|---|---|
| 6번선 | 4번선 7th · 3번선 3rd · 2번선 5th | 5번선 |
| 5번선 | 4번선 5th · 3번선 7th · 2번선 3rd | 6번선 |
| 4번선 | 3번선 5th · 2번선 7th · 1번선 3rd | 5·6번선 |
| 3번선 | 4번선 5th · 2번선 3rd · 1번선 7th | 5·6번선 |

사용자가 루트 선 버튼을 누를 때마다 해당 "공식"이 `#fournote-rule-hint` 영역에 표시됩니다.

### ⑥ 생략된 선(muted string) 시각적 구분
- `_calcFourNoteVoicing()`이 `mutedStrings` 배열을 반환하도록 확장
- `noteColorFn`에서 **사용 중인 선**(opacity 0.18)과 **생략된 선**(opacity 0.08)을 구분
- `#fournote-mute-info` 영역에 "✕ 뮤트: 5번선, 6번선" 같은 힌트 텍스트 표시

### ⑦ 색상 체계 개선 (인터벌 인지도 향상)
포 노트 섹션의 색상을 **인터벌 역할이 즉시 구분되도록** 조정:

| 역할 | 기존 | 변경 |
|---|---|---|
| R | 주황 | 주황 (유지) |
| 3 / ♭3 | 연파랑 | **녹색** (트라이어드와 통일) |
| 5 / ♭5 | 진파랑 | 진파랑 (유지) |
| 7 / ♭7 / °7 | 파랑 계열 | **보라** (3음과 명확히 구분) |

트라이어드의 3음도 녹색으로 통일하여 두 섹션 간 일관성 확보.

## 🖥️ HTML 측 필요 변경 (이 파일에는 없지만 별도 작업 필요)

수정된 JS가 기대하는 DOM 엘리먼트들:
- `#fournote-legend` — 인터벌 범례 컨테이너 (신규)
- `#fournote-rule-hint` — 루트 선별 공식 힌트 (신규)
- `#fournote-mute-info` — 뮤트 선 표시 (신규)
- `#fournote-fb`, `#fournote-fb-label` — 기존 그대로 사용

**트라이어드 섹션 HTML에서 제거해야 할 요소:**
- `[data-chordtype="7"]`, `[data-chordtype="maj7"]`, `[data-chordtype="m7"]`, `[data-chordtype="m7b5"]` 버튼
- `[data-voicing="3rd"]` 버튼 (3rd Inversion — 4음 코드에만 존재)
- `[data-strgrp="456"]` 버튼은 유지 (3화음도 4-6번줄 set 가능)

**포 노트 섹션 HTML 버튼 예시:**
```html
<button class="fn-type-btn" data-fntype="maj7">maj7</button>
<button class="fn-type-btn" data-fntype="dom7">7</button>
<button class="fn-type-btn" data-fntype="m7">m7</button>
<button class="fn-type-btn" data-fntype="m7b5">m7♭5</button>
<button class="fn-type-btn" data-fntype="dim7">dim7</button>
```
